# Source Material Intake Hygiene

This file defines cross-collection input hygiene rules for ACC content posting workflows.

## Core rule

When collecting source materials from the human, explicitly discourage duplicate resends of the same image/content.

The goal is to prevent the same underlying asset from being mistaken as multiple publishable inputs later in the workflow.

## What to remind the human

When the workflow reaches asset/material collection, explicitly remind the human:

- do not resend the same image content multiple times as if it were a new asset
- if the same image/file is being mentioned again only for explanation, extraction, emphasis, or correction, say that clearly
- repeated identical images/files can be mistaken as separate publishable assets later

## Why this matters

This is especially important when:
- a route map is first sent in a batch and later resent alone for extraction
- the same screenshot is resent while discussing details
- the same reference image is resent in different chat turns
- the same Xiaohongshu / Bilibili / YouTube / document source is pasted again as if it were a new input

## Collection-specific interpretation

### Media
Repeated images can be mistaken as separate gallery/body assets.

### Routes
Repeated route maps / screenshots / GPX-related images can be mistaken as multiple route illustrations or conflicting route inputs.

### Knowledge
Repeated screenshots / diagrams / reference images can be mistaken as multiple figures or separate source materials.

## Operational rule

Use this as an explicit intake reminder, not just an internal inference.

Prefer preventing the ambiguity at the input stage over trying to deduplicate it later through complex reasoning.
