# Asset Path and Naming Rules

Shared cross-skill asset conventions.

## Principles

- asset paths must align with ACC repo governance
- asset names must be deterministic and human-readable
- collection-specific rules may extend this file but should not contradict it

## Global naming rules

- lowercase only
- use hyphens, not spaces or underscores
- avoid UUID-like names in final repo output
- avoid `.HEIC`
- keep extensions lowercase

## Event examples

- `cover.jpg`
- `wechat-qr.png`
- `gallery/01-group-start.jpg`
- `gallery/02-cafe-stop.jpg`
