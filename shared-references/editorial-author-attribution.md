# Editorial Author Attribution

This file defines author-attribution rules for editorial-style collections.

## Core rule

For `media` and `knowledge` posting workflows, author attribution must be clarified explicitly.

Do not silently default to `ACC Club` just because the repo schema allows a default.
That repo default is a compatibility fallback, not an intake shortcut.

## Why this matters

For editorial-style content, `author` is part of the content meaning, not only a technical field.
The workflow should know whether the piece is:
- written in the club voice
- attributed to a named person
- co-authored
- compiled/edited from someone else's material

## Required intake behavior

Before a media or knowledge draft is considered publishable, clarify:
- who the author should be shown as
- whether it should be `ACC Club` or a named individual
- whether the named individual is the true author, interview subject, editor, or narrator

## Allowed fallback

If the collaborator truly wants a generic club byline, that is allowed.
But it should be an explicit decision, not an unasked assumption.

## Operational rule

Treat unresolved author attribution as a production-relevant missing field for media and knowledge workflows.
