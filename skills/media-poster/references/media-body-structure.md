# Media Body Structure

Phase-1 media posts should not be treated as raw frontmatter wrappers.
They should generate a readable ACC media page draft with a clear editorial center.

## Core rule

Decide the body structure from the **page framing**, not only from the source shape.

A link-led input may still become a recap-led page.
A gallery-heavy input may still become an adventure-led story.

## Recommended body structure families

### 1. Video-led page
Use when the video is the primary reason the page exists.

Recommended structure:
- short opening hook
- what the video is / why it matters
- watch section or embedded video context
- optional supporting stills / gallery
- optional closing note

Important runtime note:
- current ACC media detail pages can already render an embedded video from `videoUrl`
- do not duplicate the same full video embed again inside the markdown body unless there is a deliberate reason
- the body should complement the runtime embed, not blindly repeat it

### 2. Recap-led page
Use when the main goal is to retell what happened in an event / ride / recap.

Recommended structure:
- opening summary
- recap story / what happened
- highlights or key moments
- supporting media / related links
- optional closing note

### 3. Interview-led page
Use when the subject or person is the center.

Recommended structure:
- short intro to the subject
- why this interview matters
- key themes / excerpts / story points
- supporting media or source links
- closing note

### 4. Adventure-led page
Use when the journey itself is the center.

Recommended structure:
- scene-setting opening
- route / trip / experience summary
- memorable moments
- supporting media or external links
- takeaway / closing

## Body quality rules

- do not publish an empty shell that only contains one external link
- do not over-polish into empty marketing copy
- keep the body editorially coherent with the chosen framing
- if there is a related event page, reference it naturally when helpful
- if external links exist, explain their role instead of dumping them without context

## Phase-1 generation rule

If not all details are available, generate a draft with clear limited scope rather than pretending facts exist.

The body can be shorter than an event page, but it should still feel deliberate and complete enough to review.
