# Media Intake Question Flow

This file defines the Phase-1 questioning strategy for `media-poster`.

## Goal

Collect enough information to produce a real ACC media preview package without over-interrogating the human.

## Questioning principle

Ask in layers, not as a giant form dump.

User input beats artificial completeness.

If a production-critical fact is unknown or ambiguous, stop and ask immediately.
Do not invent placeholder facts just to make the draft look complete.
An explicit unknown is better than a polished fake.

## Core distinction

Separate these two decisions early:

1. **source shape**
   - asset-led
   - link-led
   - text-led
   - mixed brief

2. **editorial framing**
   - video-led
   - recap-led
   - interview-led
   - adventure-led

Do not treat them as the same decision.

### Layer 1 · Establish the media core
These are the first things to secure:
- what this piece is
- whether it belongs in the `media` collection
- the working title or working subject
- approximate date / time context
- target media type
- whether the piece should be featured or not

If these are still unclear, do not jump into asset details yet.

### Layer 2 · Secure the publishable minimum
Ask for the minimum set needed to create a credible media draft:
- one-line summary / what this piece is about
- primary asset or story anchor
- whether the page should feel video-led, recap-led, interview-led, or adventure-led
- any must-include external links
- whether there is already a cover choice

Important:
- do not assume a YouTube/Bilibili/Xiaohongshu link is enough by itself
- do not let the page become an empty shell around one link
- first ask what role the link should play in the page

### Layer 3 · Collect assets and supporting materials
Once the media core is clear, ask for:
- cover image candidate
- gallery images if relevant
- video link if relevant
- Xiaohongshu or other source/distribution link if relevant
- supporting notes, captions, recap text, or interview material

When collecting images, explicitly remind the user:
- do not resend the same image content multiple times just because it is being referenced again
- if the same image is only being resent for explanation / extraction / emphasis, say that clearly instead of sending it again as if it were a new asset
- repeated identical images can be mistaken as separate publishable assets later in the workflow

### Layer 4 · Fill enhancement fields
Only after the above:
- tags
- exact featured decision if still unclear
- multilingual nuance if needed
- body emphasis details (story first vs watch first vs subject first)

## Hard-required fields for Phase 1 draft generation

At minimum, secure:
- title or working title
- date context
- target media type
- one-line summary
- primary asset or story anchor

If one of these is missing, keep collecting before generating a meaningful draft.

## Can be deferred to revision round
These can be placeholders in an early preview if necessary:
- final cover choice
- final gallery set
- final tag set
- exact featured flag
- final external link formatting

## Preferred compact questions

When the user drops a link first, prefer a compact question such as:

- `这个你想做成视频主内容，还是做成一条活动回顾里的素材入口？`

When the user drops images first, prefer a compact question such as:

- `这些你想整理成活动回顾，还是偏图集展示？如果有同样内容的图，不要重复发。`

When the user gives recap text first, prefer a compact question such as:

- `这条你想做成纯文字回顾为主，还是还会挂视频 / 图集链接？`

## Anti-patterns

Do not:
- dump 15 questions at once
- assume source shape already determines final page framing
- produce a media page that has links but no editorial point of view
- pretend missing facts exist
- block preview just because enhancement fields are missing
